# 使用手册

## 概述
Tsunami UDP 是一个基于UDP协议的高性能数据传输系统，支持服务端和客户端模式。该系统通过优化的数据传输机制实现高效、可靠的数据通信。

## 服务端启动

服务端是数据传输的核心组件，负责监听连接请求并处理数据传输任务。

### 命令格式
```bash
./rtserver/tsunami_server \
    -p 5000 \              # 监听端口
    -w 32 \                   # 窗口大小
    -t 8                      # 工作线程数
```

### 参数说明
- `-p`: 设置服务器监听的端口号，默认为5000
- `-w`: 设置窗口大小，用于控制缓冲区大小，默认为32
- `-t`: 设置工作线程数量，用于并发处理多个连接请求，默认为8

### 启动示例
```bash
# 在后台运行服务器，监听端口5000
nohup ./rtserver/tsunami_server -p 5000 -w 32 -t 8 &
```

## 客户端连接

客户端用于发起数据传输请求，并与服务器建立连接进行数据交换。

### 命令格式
```bash
./rtclient/tsunami_client \
    -s 192.168.1.100 \     # 服务器地址
    -p 5000 \                  # 服务器端口
    -f large_file.zip          # 要传输的文件
```

### 参数说明
- `-s`: 指定要连接的服务器IP地址
- `-p`: 指定服务器监听的端口号
- `-f`: 指定要传输的文件路径

### 连接示例
```bash
# 连接到本地服务器的5000端口，传输large_file.zip文件
./rtclient/tsunami_client -s 127.0.0.1 -p 5000 -f large_file.zip
```

## 配置参数详解

### 端口设置 (-p)
- 默认值: 5000
- 取值范围: 1024~65535
- 注意事项: 确保端口未被其他程序占用

### 窗口大小 (-w)
- 默认值: 32
- 取值范围: 1~1024
- 功能作用: 控制发送和接收缓冲区的大小，影响传输效率

### 线程数 (-t)
- 默认值: 8
- 取值范围: 1~最大CPU核心数
- 影响因素: 多线程可以提高并发处理能力，但过多会增加系统开销

## 文件传输注意事项

1. **文件路径**: 请确保指定的文件路径正确且可访问
2. **网络环境**: 服务端和客户端需要在同一个局域网内或有正确的网络配置
3. **权限设置**: 传输目录需要读写权限
4. **性能优化**: 根据实际需求调整窗口大小和线程数量以获得最佳性能

## 故障排除

如果遇到连接问题，请检查以下内容:

1. 服务器是否正在运行
2. 端口是否正确开放
3. 网络连通性
4. 防火墙设置

例如，使用netstat命令查看端口状态:
```bash
netstat -an | grep 5000
```

或者使用telnet测试连接:
```bash
telnet 192.168.1.100 5000
```

## 性能调优建议

根据不同的应用场景，您可以调整以下参数来优化性能:

1. 增加工作线程数(-t)以支持更多并发连接
2. 调整窗口大小(-w)以适应不同带宽环境
3. 使用更大的端口号避免冲突

## 安全考虑

1. 仅允许可信的客户端连接到服务器
2. 对敏感数据进行加密传输
3. 设置适当的防火墙规则
